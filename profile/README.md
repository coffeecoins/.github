### Standards

All repositories in this organization follow shared engineering standards:

- **Branching**: `main` → `rc` → `staging` → `develop`, with `hotfix/*` for critical fixes
- **Commits**: [Conventional Commits](https://www.conventionalcommits.org) enforced on every PR
- **Releases**: Automated [Semantic Versioning](https://semver.org) via semantic-release
- **Security**: SAST (CodeQL + Semgrep), secret scanning, dependency audits, and DAST on every release candidate
- **Quality**: MegaLinter, 80% test coverage minimum, mandatory code review
- **Compliance**: SOC2, ISO27001, GDPR controls built into the development workflow
