# Security Policy — CoffeeCoins

This is the organization-wide security policy. Individual repositories may extend it.

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

Use one of the following private channels:

1. **GitHub Security Advisories** (preferred): Navigate to the affected repository → Security tab → Report a vulnerability
2. **Email**: security@coffeecoins.com

### What to Include

- Type of vulnerability (SQLi, XSS, auth bypass, etc.)
- Affected repository and version/branch
- Steps to reproduce
- Proof-of-concept code (if available)
- Assessed impact

## Response Timeline

| Milestone | Target |
|-----------|--------|
| Acknowledge receipt | 48 hours |
| Confirm vulnerability | 5 business days |
| Issue fix / mitigation plan | 15 business days |
| Coordinated public disclosure | 90 days |

## Compliance Targets

All CoffeeCoins repositories target compliance with **SOC2**, **ISO27001**, and **GDPR**.
